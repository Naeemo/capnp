# LTU - 内容验证方案

## 核心假设
> **3岁孩子会对特定形式的"屏幕+语音"互动内容持续参与5分钟以上**

## 验证目标
1. **内容类型**：什么主题/形式孩子最投入？
2. **互动节奏**：多长时间一个互动节点最合适？
3. **反馈形式**：语音/视觉/动作，什么组合最有效？
4. **流失节点**：孩子什么时候会离开？为什么？

---

## 验证方法：人工Wizard of Oz

### 为什么不用真实AI
- 开发ASR+LLM+TTS链条需要数周
- 早期验证不需要技术完美，需要**内容形式**正确
- 人工扮演AI，可以灵活调整策略，快速试错

### 工具准备
| 物品 | 用途 | 成本 |
|------|------|------|
| 手机/平板 | 播放内容+收集视频 | 已有 |
| 蓝牙耳机（家长戴） | 接收指令，语音回应孩子 | ¥100 |
| 简单脚本/卡片 | 提示家长说什么/做什么 | ¥0 |
| 录音笔/手机录音 | 记录完整互动过程 | 已有 |

### 角色分工
```
孩子 ←→ 屏幕（播放视频/动画）
         ↑
    家长（躲在视线外，通过耳机接收指令，语音回应孩子）
         ↑
    观察者（记录时间、反应、关键事件）
```

## 测试内容矩阵

### 维度1：内容类型（6种）

| 类型 | 描述 | 示例 |
|------|------|------|
| **A. 互动故事** | 选择分支剧情 | "小兔子要过桥，走木桥还是石头桥？" |
| **B. 问答探索** | 孩子提问，AI回应 | "这是什么？" "为什么天是蓝的？" |
| **C. 模仿游戏** | 跟着做动作 | "像我一样跳" "学小猫叫" |
| **D. 寻宝任务** | 摄像头识别物品 | "去找一个红色的东西给我看" |
| **E. 儿歌跟唱** | 音乐+简单动作 | 放歌，停顿等孩子接唱 |
| **F. 角色扮演** | 假装场景 | "你是医生，我是病人，咳嗽了" |

### 维度2：互动节奏（3档）
- **高频**：每30秒一个互动点
- **中频**：每60-90秒一个互动点
- **低频**：每3-5分钟一个互动点（主要是故事）

### 维度3：反馈形式（4种组合）
- 纯语音
- 语音+屏幕动画/图片
- 语音+家长动作示范（摄像头拍进去）
- 语音+音效/音乐反馈

## 测试流程

### 招募
- **5-8个家庭**，孩子年龄2.5-3.5岁
- 家庭有电视/大屏，家长愿意配合扮演"AI"

### 单次测试结构（20分钟）
```
0:00-2:00   暖场：简单打招呼，孩子适应
2:00-7:00   测试内容A（高频+语音+动画）
7:00-12:00  测试内容B（中频+纯语音）
12:00-17:00 测试内容C（低频+动作）
17:00-20:00 自由：孩子主导，看会发生什么
```

### 观察指标
| 指标 | 记录方式 | 成功标准 |
|------|---------|---------|
| **持续时长** | 计时器 | ≥5分钟不离开屏幕 |
| **主动参与** | 观察记录 | 主动说话/做动作 ≥3次 |
| **情绪状态** | 视频标注 | 开心/专注占比 ≥70% |
| **完成指令** | 观察记录 | 听懂并执行指令 ≥50% |
| **流失节点** | 记录时间+当时内容 | 找出共性的流失点 |

### 家长问卷（每次测试后）
1. 你觉得孩子最喜欢哪个环节？为什么？
2. 哪个环节孩子最没兴趣？
3. 你觉得节奏太快、太慢还是刚好？
4. 如果这是真实产品，你会愿意让孩子每天玩多久？
5. 你最担心的问题是什么？

## 快速测试脚本示例

### 脚本1：互动故事（高频）
```
[屏幕显示：森林背景，一只小兔子]
AI: "小兔子要去外婆家，前面有两条路。你觉得走哪条？"
    [停顿3秒，等孩子回应]
    
    IF 孩子指向左边/说左边:
        "好！我们走左边的花路！"
        [屏幕切换：花路，蝴蝶飞]
        "哇，有蝴蝶！你想数一数有几只吗？"
        
    IF 孩子指向右边/说右边:
        "右边的石头路！小心脚下哦"
        [屏幕切换：石头路，小溪流]
        "听，有流水的声音"
        
    IF 孩子没反应:
        "小兔子在等你的决定呢，左边还是右边？"
        [重复一次，还没反应就默认选左边继续]

[每30秒一个选择点，持续5分钟]
```

### 脚本2：寻宝任务（中频）
```
AI: "我看到你手里有个东西！举起来让我看看是什么颜色的？"
    [孩子举起玩具]
    "红色的！圆圆的！是球吗？"
    
    [孩子确认]
    "太棒了！现在能帮我找一个绿色的东西吗？"
    [孩子去找]
    "找到了！那是什么？"
    
    [孩子展示]
    "树叶！绿色的树叶！你闻闻香不香？"
```

### 脚本3：模仿游戏（低频+动作）
```
[屏幕显示：小动物做动作]
AI: "我是小青蛙，呱呱！这样跳！"
    [家长做跳跃动作，入镜]
    "你能像我一样跳吗？"
    [等待孩子跳]
    "跳得好高！现在我是小蛇，嘶嘶，在地上爬"
    [做爬行动作]
    "你来试试？"
```

## 成功标准

### 核心指标（必须满足）
- **≥60%的测试**中，孩子在单个内容类型上持续≥5分钟
- **≥1种内容类型**的平均持续时长≥8分钟
- **家长反馈**："我愿意让孩子每天用这个15-20分钟"

### 辅助指标（帮助优化）
- 找出表现最好的"内容类型+节奏+反馈"组合
- 明确流失原因Top3
- 收集家长自发提出的功能需求

## 可能的风险

| 风险 | 应对 |
|------|------|
| 孩子不配合，一直乱跑 | 缩短单次测试到10分钟，增加暖场时间 |
| 家长扮演不自然 | 提供详细话术卡，预演1次 |
| 孩子发现是家长在说话 | 家长躲在屏风后，使用变声器App |
| 样本量太小，结论不可靠 | 明确这是定性验证，非定量；如有必要，第二阶段扩大样本 |

## 下一步决策点

测试完成后，根据结果选择：

**A. 内容方向明确，进入技术验证**
- 开发真实ASR+TTS Demo
- 优化特定内容类型的交互

**B. 内容方向不明确，再迭代**
- 重新设计内容形式
- 再跑一轮小规模测试

**C. 发现致命问题，项目暂停**
- 3岁孩子根本不对屏幕内容产生互动
- 家长强烈抗拒任何形式的"替代陪伴"
