# LTU - 交互与启动设计

## 启动触发：机械封盖开关

### 决策
**物理摄像头封盖作为电源开关**
- 封盖打开 → 设备启动，进入互动模式
- 封盖关闭 → 设备断电/休眠，摄像头物理遮蔽

### 为什么这个方案最优
| 方案 | 问题 | 封盖方案的优势 |
|------|------|---------------|
| 语音唤醒 | 需始终监听，隐私敏感 | 无后台监听，物理断连 |
| 定时启动 | 机械，无法响应孩子当下需求 | 孩子主动控制，有掌控感 |
| 摄像头识别 | 需持续监控，隐私不可接受 | 用户明确知道"开了才看" |
| 遥控器 | 孩子按不动，易丢失 | 孩子可自己操作封盖 |

### 3岁孩子的理解成本
封盖开关是**物理可见的、可逆的、有反馈的**操作：
- 打开盖子 → 听到启动音效/看到指示灯 → 知道"它活了"
- 关上盖子 → 灯灭 → 知道"它睡了，不再看我"
- 建立"隐私=物理遮蔽"的直观认知

### 硬件形态示意
```
    [ 电视屏幕 ]
         |
    [ USB摄像头 ]
    ┌─────────────┐
    │  ◉ 镜头    │  ← 机械封盖（滑盖/旋转盖）
    │  [====]    │     孩子可手动操作
    │   指示灯   │  ← 打开时亮起，关闭时熄灭
    └─────────────┘
         |
    [ 麦克风阵列 ]
```

### 状态机
```
封盖关闭 ──→ 硬件断电/休眠
    ↑           ↓
封盖打开 ←── 启动完成
    ↓
待机模式（等孩子说话/动）
    ↓
互动中（内容播放/对话/游戏）
    ↓
空闲超时（5分钟无互动）→ 回到待机
```

## 家长端：PC Web 优先

### 决策
**第一阶段：PC Web 管理端**
**第二阶段：小程序（验证有效后）**

### 为什么先PC Web
| 维度 | PC Web 的优势 |
|------|--------------|
| **开发速度** | 无需适配多机型，无需过审，迭代快 |
| **功能完整** | 大屏看数据、设置参数体验更好 |
| **验证阶段** | 早期用户（愿意尝试的家长）通常有电脑 |
| **技术栈** | 与电视端可复用Web技术（如果是OTT Web应用） |

### 家长端核心功能
```
登录页
  └── 仪表盘
       ├── 今日概览
       │    ├── 互动时长
       │    ├── 内容类型分布（故事/游戏/问答）
       │    └── 情绪记录（开心/无聊/烦躁）
       ├── 内容管理
       │    ├── 内容库浏览（启用/禁用特定内容）
       │    ├── 自定义上传（家长录自己的故事）
       │    └── 每日限额设置
       ├── 实时查看
       │    ├── 当前状态（待机/互动中）
       │    ├── 语音对讲（"妈妈还在开会，再等我10分钟"）
       │    └── 截图查看（非视频流，隐私考虑）
       └── 成长报告（周/月）
            ├── 词汇量增长（记录了哪些新词）
            ├── 兴趣变化（最近爱问什么类型的问题）
            └── 建议（"最近对动物感兴趣，建议去动物园"）
```

## 双用户角色的设计冲突

### 核心矛盾
| 孩子想要 | 家长想要 |
|---------|---------|
 好玩、随时回应、无限制 | 有营养、可控制、有记录 |
| 无目的探索 | 有成长目标 |
| 隐私（不想被监控） | 知情权（想知道孩子在干嘛） |

### 设计原则
**"孩子感觉自由，家长拥有控制"**

1. **孩子端无设置界面** —— 所有控制藏在校家长端
2. **内容分级不阻断** —— 超出年龄的内容自动降级（复杂词换简单词），而非"你不能看"
3. **家长介入柔和化** —— 对讲功能先问"妈妈想跟你说话，要听听吗？"，而非直接打断
4. **数据展示去焦虑** —— 不说"今天只学了5分钟"，说"今天认识了3个新动物"

## 待继续思考：MVP验证

### 可能的验证路径
**路径A：技术验证优先**
- 目标：证明"电视+摄像头+童声识别"技术可行
- 方式：开发最小Demo，找5个孩子测试语音识别准确率

**路径B：内容验证优先**
- 目标：证明"什么样的互动内容3岁孩子愿意持续参与"
- 方式：用现有工具（手机+简单App）模拟，观察孩子反应

**路径C：付费意愿验证优先**
- 目标：证明家长愿意为"替代陪伴"付费
- 方式：Landing page + 预售/问卷，看转化率
