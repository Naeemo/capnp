# LTU - Learn To Understand

> 面向3岁儿童的AI陪伴与启蒙教育

## 项目核心理念

### 为什么是3岁？
- **语言爆发期**：词汇量从200-300激增到1000+
- **认知飞跃期**：开始理解因果、分类、序列
- **社交敏感期**：需要大量互动确认自我边界
- **习惯塑形期**：注意力模式、情绪调节的基础建立

### 关键约束
| 维度 | 3岁儿童的现实 | 技术适配挑战 |
|------|--------------|-------------|
| 输入 | 不识字、不会打字、鼠标操作困难 | 语音+触摸+图像识别为主 |
| 输出 | 句子短、词汇简单、情绪优先于逻辑 | 多模态反馈（语音+视觉+情感） |
| 注意力 | 3-15分钟切换、易被视觉/声音吸引 | 节奏快、互动密度高、随时可暂停 |
| 安全 | 无法辨别有害内容、模仿性强 | 绝对的内容过滤+行为边界 |

## 可能的产品形态

### 形态A：语音伴侣（智能音箱升级版）
**核心交互**：说话 + 听故事/对话
**场景**：睡前故事、问答探索、情绪安抚
**技术栈**：ASR(童声优化) → LLM → TTS(温暖声线)

### 形态B：平板游戏化学习
**核心交互**：触摸 + 视觉反馈 + 简单语音
**场景**：形状颜色认知、数字启蒙、习惯养成
**技术栈**：视觉识别 + 游戏引擎 + 对话系统

### 形态C：实体玩具（AI玩偶）
**核心交互**：拥抱/触摸 + 对话 + 表情反馈
**场景**：全天候陪伴、分离焦虑缓解、角色扮演
**技术栈**：嵌入式语音 + 机械表情 + 云端大脑

### 形态D：家长协作工具
**核心交互**：家长设置目标，AI执行陪伴
**场景**：家长忙时接管、记录成长数据、个性化推荐
**技术栈**：家长端App + 儿童端语音设备 + 数据看板

## 待深入问题

1. **语音识别的童声难题**
   - 3岁儿童发音不清、语法混乱
   - 现有ASR模型训练数据多为成人
   - 是否需要端到端的方言/童声微调？

2. **内容安全的绝对性**
   - LLM可能生成"告诉孩子去厨房拿刀"这类内容
   - 如何在系统层面而非提示层面阻断风险？
   - 需要白名单机制还是多层过滤？

3. **情感连接 vs 技术依赖**
   - 孩子会把AI当朋友，如何处理分离？
   - 家长担心"孩子只听AI的，不听我的"
   - 如何设计让家长始终处于核心位置？

4. **商业模式**
   - 硬件售卖？订阅制？内容付费？
   - 教育资质要求？医疗/心理边界？
